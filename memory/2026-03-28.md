## 📚 学习收获

### 1. Docker Desktop 启动问题排查
**问题现象**：Docker 服务一直显示为 Stopped 状态

**解决方法**：
- 使用 `Start-Process` 启动 Docker Desktop 进程
- 使用 `docker version` 命令验证 Docker API 可用性
- 等待 Docker Desktop 完全启动（约 15-20 秒）

**经验教训**：
- PowerShell 的 `Start-Service` 可能不适用于 Docker Desktop
- 应该使用 `Start-Process` 启动 GUI 应用
- 需要等待 Docker Desktop 完全启动后才能使用 `docker` 命令

### 2. Docker 容器环境变量配置
**问题现象**：环境变量 `VECTOR_STORE` 显示为 `${VECTOR_STORE:-chroma}`，没有被正确解析

**解决方法**：
- 使用 `docker run` 命令直接启动容器，并设置环境变量
- 通过 `-e` 参数传递环境变量：`-e VECTOR_STORE=chroma`
- 验证环境变量是否正确设置：`docker inspect container_name --format='{{json .Config.Env}}'`

**经验教训**：
- Docker Compose 的环境变量解析有时会出现问题
- 直接使用 `docker run` 命令可以更精确地控制环境变量
- 需要定期检查容器的环境变量配置

### 3. 容器网络配置
**问题现象**：Web 容器无法访问 API 容器

**解决方法**：
- 使用 `--link` 参数建立容器之间的链接
- 设置正确的 API URL：`-e CONSOLE_API_URL=http://docker-api-1:5001`
- 验证容器网络：`docker network inspect container_network`

**经验教训**：
- 容器之间需要通过容器名称（如 `docker-api-1`）而不是主机名来访问
- 需要确保所有容器在同一个网络中
- 端口映射也很重要，需要确保 API 容器的 5001 端口被映射到主机

### 4. 容器状态管理
**问题现象**：容器名称冲突，无法重复启动

**解决方法**：
- 使用 `docker stop` 停止容器
- 使用 `docker rm -f` 删除容器
- 使用 `docker ps -a` 查看所有容器（包括已停止的）

**经验教训**：
- 启动容器前需要先删除旧容器
- 使用 `-f` 参数强制删除运行中的容器
- 定期清理不需要的容器，避免名称冲突

## 🐛 遇到的问题

### 问题1：Docker Desktop 无法启动
**现象**：Docker 服务状态为 Stopped，`docker ps` 命令无法执行

**解决方法**：
1. 使用 `Start-Process "C:\Program Files\Docker\Docker\Docker Desktop.exe"` 启动 Docker Desktop
2. 等待 15-20 秒，让 Docker Desktop 完全启动
3. 使用 `docker version` 命令验证 Docker API 可用

**经验教训**：
- PowerShell 的 `Start-Service` 不适用于 Docker Desktop
- 应该使用 `Start-Process` 启动 GUI 应用
- 需要等待 Docker Desktop 完全启动

### 问题2：Web 界面无法访问
**现象**：访问 `http://localhost:3000` 显示空白界面

**解决方法**：
1. 检查 Web 容器日志，确认容器是否正常启动
2. 检查 API 容器日志，确认 API 是否正常运行
3. 检查环境变量配置，确保 API URL 设置正确
4. 使用 `--link` 参数建立容器之间的链接
5. 重新部署 Web 容器，设置正确的 API URL

**经验教训**：
- Web 容器需要通过容器名称（`docker-api-1`）访问 API，不是 `127.0.0.1`
- 需要设置正确的环境变量：`CONSOLE_API_URL=http://docker-api-1:5001`
- 端口映射很重要，需要确保 API 容器的 5001 端口被映射

### 问题3：环境变量未被正确解析
**现象**：环境变量 `VECTOR_STORE` 显示为 `${VECTOR_STORE:-chroma}`，Dify 无法识别 ChromaDB

**解决方法**：
1. 使用 `docker run` 命令直接启动容器，并通过 `-e` 参数设置环境变量
2. 验证环境变量：`docker inspect container_name --format='{{json .Config.Env}}' | ConvertFrom-Json | Where-Object { $_ -like "*VECTOR_STORE*" }`

**经验教训**：
- Docker Compose 的环境变量解析有时会出现问题
- 直接使用 `docker run` 可以更精确地控制环境变量
- 需要定期检查容器的环境变量配置
