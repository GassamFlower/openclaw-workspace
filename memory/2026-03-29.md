## 部署时间线

### 1. 初始部署尝试
**问题**: 网页打开空白
**原因**: 容器网络配置错误
**解决**: 添加数据库容器到 docker_default 网络

### 2. 数据库迁移失败
**问题**: API 服务无法初始化，数据库表为空
**根本原因**: 环境变量缺少 `MIGRATION_ENABLED=true`
**解决**: 在 docker-compose.yaml 中添加 `MIGURATION_ENABLED: "true"`

### 3. API URL 配置错误
**问题**: Web 容器无法通过 `localhost:5001` 访问 API
**根本原因**: 容器内部的 `localhost` 指向容器自己，而不是 API 容器
**解决**: 将环境变量改为 `http://api:5001`，使用 Docker 网络中的容器名称

### 4. CORS 错误
**问题**: 浏览器控制台显示 CORS 错误
**根本原因**: API 没有配置允许跨域请求
**解决**: 添加环境变量 `CONSOLE_CORS_ALLOW_ORIGINS: "*"` 和 `WEB_API_CORS_ALLOW_ORIGINS: "*"`

### 5. Web 容器环境变量硬编码
**问题**: Web 容器启动时使用硬编码的 `127.0.0.1:5001`
**根本原因**: Docker 镜像内部有硬编码的默认环境变量
**尝试的解决方案**:
- ✗ 修改 .env 文件
- ✗ 修改 docker-compose 配置
- ✗ 使用 command 覆盖启动命令
- ✗ 使用 entrypoint 覆盖启动命令
**最终解决方案**: 使用官方的 docker-compose 配置文件并正确生成

## 遇到的关键问题

### 问题 1: 容器网络隔离
**症状**: API 容器和数据库容器在错误的网络中
**错误信息**:
```
psycopg2.OperationalError: could not translate host name "db_postgres" to address: Name or service not known
```
**解决**: 添加数据库容器到 docker_default 网络
```bash
docker network connect docker_default docker-db_postgres-1
```

### 问题 2: 数据库未初始化
**症状**: Web 页面空白，API 无法工作
**错误信息**: 没有数据库表
**解决**: 添加 `MIGRATION_ENABLED: "true"` 到环境变量

### 问题 3: 跨域访问被阻止
**症状**: 浏览器控制台 CORS 错误
**解决**: 添加 CORS 环境变量

### 问题 4: 网络配置
**症状**: `127.0.0.1:5001` 在容器内部无法访问
**解决**: 使用容器名称 `api:5001` 代替 `localhost:5001`

## 经验教训

### 1. 容器间通信
- 容器内部的 `localhost` 不等于宿主机的 `localhost`
- 必须使用 Docker 网络中的容器名称进行通信

### 2. 数据库迁移
- 必须显式启用 `MIGRATION_ENABLED=true`
- 首次部署时数据库迁移是必要的

### 3. CORS 配置
- API 服务需要配置 CORS 允许前端访问
- 环境变量 `CONSOLE_CORS_ALLOW_ORIGINS` 和 `WEB_API_CORS_ALLOW_ORIGINS` 很重要

### 4. Docker 镜像
- 官方镜像可能有硬编码的默认环境变量
- 使用官方的 docker-compose 配置文件更可靠
- 不要尝试覆盖镜像内部的环境变量

### 5. VPN 影响
- VPN 可能会影响 Docker 网络通信
- 如果遇到网络问题，尝试关闭 VPN
