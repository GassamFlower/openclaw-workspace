## 学习收获和经验教训

### 1. Docker Compose 配置
- **问题**：YAML 语法错误导致服务无法启动
- **解决**：使用 `Select-String` 命令定位错误行
- **教训**：定期检查 YAML 配置的语法正确性

### 2. 容器网络配置
- **问题**：容器无法解析主机名
- **解决**：确保所有服务在同一网络，使用容器名称而非 localhost
- **教训**：Docker 网络通信必须使用容器名称

### 3. 环境变量配置
- **问题**：环境变量未正确传递到容器
- **解决**：在 `.env` 文件中明确定义所有必要变量
- **教训**：`.env` 文件是配置的核心，必须完整

### 4. 数据库迁移
- **问题**：首次部署时数据库未初始化
- **解决**：设置 `MIGRATION_ENABLED: "true"` 环境变量
- **教训**：数据库迁移需要显式启用

### 5. 简化部署的重要性
- **发现**：官方完整配置过于复杂，容易出错
- **解决**：创建最小化配置，只包含核心服务
- **教训**：对于学习或测试环境，简化配置更可靠
