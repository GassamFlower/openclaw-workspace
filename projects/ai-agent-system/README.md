# AI Agent System - 智能体运营系统

> **项目简介**：基于多智能体架构的企业级运营中枢系统
>
> **总周期**：10-12周（4个阶段）
> **技术栈**：Dify, FastAPI, LangGraph, CrewAI, React/Vue, Redis/PostgreSQL
> **项目目标**：打造门神文化 + 景区 + 研学 + 电商的一体化智能运营平台

## 🎯 四大阶段概览

### 阶段一：基础设施与知识库搭建（1-3周）⭐
- 目标：跑通主干流程，智能客服具备基本问答能力
- 核心产出：知识库系统、可文字对话的小程序测试版

### 阶段二：LangGraph中枢与核心业务流打通（4-6周）⭐⭐
- 目标：引入状态管理，票务与研学业务闭环
- 核心产出：支持复杂上下文对话、能推送购票/研学卡片的小程序正式版

### 阶段三：内容营销与多渠道扩展（7-9周）⭐⭐⭐
- 目标：释放运营人力，全域流量覆盖
- 核心产出：内容自动化生产工作流、抖音AI客服、电商导购服务

### 阶段四：监控管理与平稳交付（10-12周）⭐⭐⭐⭐
- 目标：数据可视化，完善异常处理机制
- 核心产出：完整交付整个多智能体运营中枢、管理后台、实施与运维手册

## 📚 文档说明

### 主目录文档（docs/）
所有不随开发阶段变动的通用文档都放在 `docs/` 目录中，包括：
- 架构设计文档
- API接口文档
- 部署和运维手册
- 测试报告

### 阶段文档（stageX/）
每个阶段的文档和代码都放在对应的 `stageX/` 目录中：
- 阶段一的文档放在 `stage1/`
- 阶段二的文档放在 `stage2/`
- 阶段三的文档放在 `stage3/`
- 阶段四的文档放在 `stage4/`

每个阶段目录下都会有一个 `README.md`，说明该阶段的目标、任务和产出物。

## 🔧 开发流程

1. **阶段规划**：根据阶段目标制定详细任务清单
2. **文档先行**：每个阶段开始前，先编写技术文档
3. **开发实现**：按照任务清单逐步实现功能
4. **测试验证**：每个功能完成后进行测试
5. **文档更新**：更新相关文档和README
6. **阶段交付**：阶段完成后提交该阶段的文档和代码

## 📝 注意事项

- 文档命名规范：使用英文小写字母、下划线或连字符
- 代码命名规范：遵循PEP8（Python）/ ESLint（JavaScript/TypeScript）
- Git提交规范：使用语义化提交信息（如：feat: 新增票务查询工具）
- 文档版本控制：每个阶段完成后创建新的文档版本
