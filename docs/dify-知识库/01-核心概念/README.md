# 01 - 核心概念

> Dify 的核心思想，理解这些才能用好它

## 什么是 Dify？🎯

简单说：**一个开源的 AI 应用开发平台**

- **目标**：让你能快速创建、部署和管理 AI 应用
- **核心**：可视化工作流设计 + 知识库管理 + 模型集成
- **特点**：
  - 开源免费（Self-host）
  - 可视化设计（拖拽式工作流）
  - 知识库支持（RAG 文档检索）
  - 多模型支持（OpenAI、Claude、本地模型等）
  - API 集成方便

**谁适合用？**
- 开发者
- 产品经理
- AI 应用爱好者
- 需要快速构建 AI 产品的团队

---

## 它是如何工作的？

```
用户输入 → 工作流 → 知识库检索 → 模型生成 → 输出结果
```

### 关键组件

1. **应用（App）**
   - 聊天机器人、工作流应用
   - 支持多种类型：聊天、工作流、API 等

2. **工作流（Workflow）**
   - 可视化流程设计
   - 支持条件判断、循环、迭代
   - 连接各种节点（LLM、知识库、工具等）

3. **知识库（Knowledge）**
   - 文档上传和管理
   - 自动分块和向量化
   - 智能检索

4. **节点（Nodes）**
   - LLM：调用大语言模型
   - 知识库检索：从文档中查找相关信息
   - HTTP 请求：调用外部 API
   - 代码执行：运行 Python/JS
   - 触发器：定时、Webhook 等

5. **模型提供商（Model Provider）**
   - OpenAI、Claude、本地模型等
   - 统一接口，方便切换

## 核心能力一览

| 功能 | 说明 | 例子 |
|------|------|------|
| **可视化设计** | 拖拽式工作流设计 | 从 A 节点拖到 B 节点 |

## 为什么选择 Dify？

### 对比其他平台

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Dify（自托管）** | 完全开源、免费<br>可视化设计<br>数据在自己手里 | 需要自己维护 |
| **Dify Cloud** | 开箱即用<br>无需维护 | 需要付费 |
| **LangChain** | 灵活、强大 | 配置复杂、门槛高 |
| **自写代码** | 完全控制 | 开发周期长 |

### 适用场景

✅ **适合**：
- 快速构建 AI 应用
- 需要知识库功能
- 团队协作开发
- 数据隐私要求高

❌ **不适合**：
- 只需要简单聊天功能
- 没有技术能力部署
- 对可视化设计不感兴趣

## 核心概念速查

| 概念 | 简单解释 |
|------|----------|
| **App** | 你的 AI 应用（聊天机器人、工作流） |
| **Workflow** | 工作流，应用的执行流程 |
| **Node** | 工作流中的节点（LLM、工具等） |
| **Knowledge** | 知识库，存储文档 |
| **Chunk** | 文档分块，知识库的基本单位 |
| **Embedding** | 向量化，用于检索 |
| **Pipeline** | 知识库处理流程 |
| **Plugin** | 插件，扩展功能 |
