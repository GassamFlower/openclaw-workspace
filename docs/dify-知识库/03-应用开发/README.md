# 03 - 应用开发

> 创建、配置、调试 Dify 应用

## 🎯 应用类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **聊天机器人** | 简单的对话应用 | 客服、助手 |
| **工作流应用** | 复杂流程 | 多步骤任务 |
| **API** | 提供 HTTP 接口 | 集成到其他系统 |
| **Chatflow** | 流式对话 | 交互式应用 |

---

## 📝 创建应用

### 步骤

1. 登录 Dify
2. 点击 "创建应用"
3. 选择类型
4. 填写信息
5. 点击 "创建"

### 基础信息

- **名称**：应用名称
- **描述**：应用说明
- **标签**：分类标签

---

## ⚙️ 配置模型

### 添加模型提供商

1. 左侧菜单 → "模型提供商"
2. 点击 "添加提供商"
3. 选择提供商类型
4. 填写配置

### 支持的模型提供商

| 提供商 | 说明 |
|--------|------|
| **OpenAI** | GPT-4、GPT-3.5 等 |
| **Anthropic** | Claude 3.5 Sonnet 等 |
| **Ollama** | 本地模型 |
| **vLLM** | 本地模型 |
| **其他** | 自定义模型 |

### 配置示例

```json
{
  "name": "OpenAI",
  "api_key": "sk-xxx",
  "base_url": "https://api.openai.com/v1",
  "models": ["gpt-4o", "gpt-3.5-turbo"]
}
```---

## 🔄 工作流设计

### 基础工作流

```
用户输入 → LLM → 回复
```

### 常用节点

| 节点 | 功能 | 使用场景 |
|------|------|----------|
| **LLM** | 调用模型 | 生成回复 |
| **知识库检索** | 检索文档 | RAG |
| **HTTP 请求** | 调用 API | 外部服务 |
| **代码** | 执行代码 | 数据处理 |
| **If-Else** | 条件判断 | 路由 |
| **循环** | 重复执行 | 批量处理 |
| **参数提取** | 提取结构化数据 | 复杂任务 |

### 连接节点

1. 拖拽节点到画布
2. 点击节点 A 的输出点
3. 连接到节点 B 的输入点

---

## 🧪 测试应用

### 本地测试

1. 点击页面顶部的 "试运行" 按钮
2. 输入测试问题
3. 查看结果

### API 测试

```bash
curl -X POST http://your-dify-instance/v1/chat-messages \
  -H "Authorization: Bearer YOUR-API-KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": {},
    "query": "测试问题",
    "response_mode": "blocking",
    "conversation_id": ""
  }'
```---

## 📤 发布应用

### Web App

1. 点击 "发布"
2. 选择 "Web App"
3. 配置选项
4. 点击 "发布"

### API

1. 点击 "发布"
2. 选择 "API"
3. 复制 API 端点

### 嵌入到网站

```html
<iframe src="https://your-dify-instance/embed" width="100%" height="600"></iframe>
```---

## 🔍 调试应用

### 查看日志

1. 左侧菜单 → "调试"
2. 点击 "运行单次"
3. 查看节点执行详情

### 常见问题

| 问题 | 解决方法 |
|------|----------|
| AI 不回复 | 检查 LLM 配置 |
| 回复不准确 | 优化提示词 |
| 工作流中断 | 查看节点错误 |

---

## 📊 监控应用

### 性能监控

1. 左侧菜单 → "监控"
2. 查看使用数据
3. 分析用户反馈

### 反馈收集

- ✅ 👍 点赞/点踩
- 💬 文本反馈
- ⏱️ 响应时间

---

## 🎨 优化建议

### 1. 提示词优化

✅ **清晰明确**
```
请以 Markdown 格式回复
```❌ **模糊不清**
```
回复一下
```

### 2. 知识库优化

- ✅ 文档质量：清晰、准确
- ✅ 分块策略：合理的分块大小
- ✅ 检索设置：相关度、召回数量

### 3. 工作流优化

- ✅ 减少不必要的节点
- ✅ 使用缓存（如适用）
- ✅ 优化条件判断
