# 05 - 工作流

> 可视化设计 AI 工作流，实现复杂任务自动化

## 🎯 工作流概览

工作流是 Dify 的核心功能，通过可视化节点设计实现：

- 🧩 拖拽式设计
- 🔄 条件判断和循环
- 🔗 连接多个节点
- 📊 复杂业务逻辑

---

## 📝 创建工作流

### 步骤

1. 创建应用（选择 "工作流" 类型）
2. 进入 "工作流" 页面
3. 开始设计

---

## 🧩 节点类型

### 1. LLM 节点

**功能**：调用大语言模型生成内容

**配置**：
```json
{
  "model": "gpt-4o",
  "prompt": "回复用户的问题：{{query}}",
  "max_tokens": 1024
}
```### 2. 知识库检索节点

**功能**：从知识库中检索相关内容

**配置**：
```json
{
  "knowledge_base": "知识库 ID",
  "top_k": 3,
  "score_threshold": 0.7
}
```### 3. HTTP 请求节点

**功能**：调用外部 API

**配置**：
```json
{
  "method": "GET",
  "url": "https://api.example.com/data",
  "headers": {
    "Authorization": "Bearer token"
  }
}
```### 4. 代码节点

**功能**：执行 Python 或 JavaScript

**配置**：
```javascript
// Python
import json

def main(query, context):
    result = {"query": query, "count": len(context)}
    return result
```### 5. If-Else 节点

**功能**：条件判断和路由

**配置**：
```json
{
  "conditions": [
    {
      "variable": "result.score",
      "operator": ">",
      "value": 0.8
    }
  ]
}
```### 6. 循环节点

**功能**：重复执行任务

**配置**：
```json
{
  "max_iterations": 10,
  "break_condition": "item.processed"
}
```### 7. 参数提取节点

**功能**：从自然语言提取结构化数据

**输出**：
```json
{
  "name": "张三",
  "age": 25
}
```### 8. 问题分类节点

**功能**：智能分类问题

**分类**：
- 技术问题
- 一般咨询
- 投诉建议

---

## 🔗 节点连接

### 连接规则

1. 节点有输出点（通常在右侧）
2. 点击输出点
3. 连接到下一个节点的输入点

### 数据传递

```json
// 节点 A 输出
{
  "query": "你好",
  "context": ["知识1", "知识2"]
}

// 节点 B 输入（变量引用）
{
  "query": "{{query}}",
  "retrieved_docs": "{{context}}"
}
```---

## 🧪 测试工作流

### 单步测试

1. 点击 "运行单次"
2. 输入输入参数
3. 查看每个节点的输出

### 完整测试

1. 点击 "试运行"
2. 输入完整输入
3. 查看最终结果

---

## 🔍 调试技巧

### 查看节点输出

1. 点击节点
2. 查看 "调试信息"
3. 检查输入输出数据

### 常见问题

| 问题 | 解决方法 |
|------|----------|
| 节点不执行 | 检查连接是否正确 |
| 数据格式错误 | 验证数据类型 |
| 条件判断失败 | 检查条件表达式 |

---

## 📊 工作流示例

### 客服机器人工作流

```
用户输入 → 问题分类 → 知识库检索 → LLM → 回复
```

### 智能助手工作流

```
用户输入 → 参数提取 → 意图识别 → 知识库检索 → LLM → HTTP API → 回复
```

### 报表生成工作流

```
定时触发 → 数据获取 → 处理 → LLM → 发送邮件
```

## 💡 最佳实践

### 1. 保持简单

- ✅ 减少不必要的节点
- ✅ 优先使用 LLM
- ❌ 避免过度复杂

### 2. 条件判断

- ✅ 明确条件逻辑
- ✅ 提供默认分支
- ❌ 避免无限循环

### 3. 错误处理

- ✅ 使用 If-Else 处理异常
- ✅ 提供友好的错误信息
- ❌ 不要忽略错误

### 4. 性能优化

- ✅ 使用缓存（如适用）
- ✅ 合理设置超时
- ❌ 避免重复调用
