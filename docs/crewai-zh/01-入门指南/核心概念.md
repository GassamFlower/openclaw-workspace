# CrewAI 核心概念

> 掌握这些核心概念,你就理解了 CrewAI 的工作原理

## 🎯 必知必会概念

### 1. Agent (智能体)

**Agent = AI 专家**

每个 Agent 都是一个独立的 AI,有自己的角色、目标和技能。

#### Agent 属性

```python
from crewai import Agent

agent = Agent(
    role="研究员",        # 角色: 在团队中的身份
    goal="研究最新技术",    # 目标: 需要达成什么
    backstory="你是一位科技专家",  # 背景: Agent 的专业背景
    verbose=True,         # 详细模式: 显示思考过程
    memory=True,          # 记忆: 是否记住对话历史
    allow_delegation=False # 是否可以委派任务
)
```

#### Agent 能力扩展

Agent 有 5 种扩展能力:

```python
agent = Agent(
    tools=[...],        # 1. Tools: 使用外部工具
    mcps=[...],         # 2. MCPs: 连接 MCP 服务器
    apps=[...],         # 3. Apps: 调用应用程序
    skills=[...],       # 4. Skills: 使用技能包
    knowledge=[...]     # 5. Knowledge: 访问知识库
)
```

### 2. Task (任务)

**Task = 工作描述**

每个 Agent 需要完成的任务。

#### Task 属性

```python
from crewai import Task

task = Task(
    description="总结最新的 AI 进展",  # 任务描述
    expected_output="一份 500 字的技术总结",  # 期望输出
    agent=researcher,  # 分配给谁
    tools=[search_tool],  # 需要用到的工具
    async_execution=False,  # 是否异步执行
    callback=save_result  # 执行回调函数
)
```

#### Task 类型

| 类型 | 说明 | 使用场景 |
|------|------|---------|
| **Standard Task** | 标准任务 | 大多数情况 |
| **Code Execution Task** | 代码执行 | 需要 Agent 运行代码 |
| **Human Feedback Task** | 人工反馈 | 需要人类介入 |

---

### 3. Crew (团队)

**Crew = 团队管理**

多个 Agent 组成的团队,负责协调任务。

#### Crew 属性

```python
from crewai import Crew

crew = Crew(
    agents=[researcher, analyst, writer],  # 团队成员
    tasks=[task1, task2, task3],  # 任务列表
    process="sequential",  # 执行流程
    verbose=True,  # 详细模式
    memory=True,  # 共享记忆
    max_iter=10,  # 最大迭代次数
    cache=True  # 结果缓存
)
```

#### Crew 执行流程

```
┌──────────┐
│  kickoff  │ (启动团队)
└────┬─────┘
     │
     ▼
┌──────────┐
│ Process  │ (根据流程执行)
└────┬─────┘
     │
     ▼
┌──────────┐
│  Task 1  │ → Agent 1 执行
└────┬─────┘
     │
     ▼
┌──────────┐
│  Task 2  │ → Agent 2 执行
└────┬─────┘
     │
     ▼
┌──────────┐
│  Task N  │ → Agent N 执行
└────┬─────┘
     │
     ▼
┌──────────┐
│   Result  │ (汇总结果)
└──────────┘
```

### 4. Process (流程)

**Process = 执行方式**

Crew 有三种执行流程:

#### Sequential (顺序执行)

```python
# 任务按顺序执行: Task 1 → Task 2 → Task 3
crew = Crew(
    agents=[agent1, agent2, agent3],
    tasks=[task1, task2, task3],
    process="sequential"  # 默认模式
)
```

**特点**:
- 严格按顺序执行
- 每个任务都等待前一个完成
- 简单直接

#### Hierarchical (层级管理)

```python
# 添加 Manager Agent
manager = Agent(
    role="项目经理",
    goal="协调团队完成目标",
    backgound="你擅长管理和协调",
    allow_delegation=True  # 允许委派
)

crew = Crew(
    agents=[manager, agent1, agent2],
    tasks=[task1, task2, task3],
    process="hierarchical"
)
```

**特点**:
- Manager Agent 调度任务
- Worker Agents 执行任务
- 更灵活,但更复杂

#### Hybrid (混合模式)

```python
# 结合顺序和层级
crew = Crew(
    agents=[...],
    tasks=[...],
    process="hybrid"  # 根据情况自动选择
)
```

**特点**:
- 简单任务顺序执行
- 复杂任务交给 Manager
- 灵活性最高

**Tool = 外部功能**

Agent 可以调用工具来执行具体操作。

### 6. Memory (记忆)

**Memory = 上下文记忆**

让 Agent 记住之前的对话。

#### Memory 类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **Buffer Memory** | 完整对话历史 | 短对话 |
| **Buffer Window** | 最近 N 轮 | 长对话 |
| **Summary Memory** | 对话摘要 | 超长对话 |

```python
from crewai.memory import BufferMemory

crew = Crew(
    agents=[...],
    tasks=[...],
    memory=BufferMemory()
)
```

---

### 7. Knowledge (知识库)

**Knowledge = 可访问信息**

Agent 可以访问的知识库。

#### 知识库类型

```python
# 1. 文档知识库
from crewai.knowledge import KnowledgeBase

kb = KnowledgeBase.from_directory("./docs")

# 2. 向量数据库
from crewai.knowledge import VectorStoreKnowledgeBase
kb = VectorStoreKnowledgeBase(embeddings=EmbeddingModel())

# 3. 文件知识
kb = FileKnowledgeBase(path="./data")
```

### 8. Flows (工作流)

**Flows = 结构化流程**

更复杂的工作流编排,支持状态管理。

```python
from crewai import Flow

flow = Flow(
    name="数据处理流程",
    description="从数据采集到报告生成",
)

# 添加节点
flow.add_node("采集", data_collection_node)
flow.add_node("处理", processing_node)
flow.add_node("分析", analysis_node)

# 添加边
flow.add_edge("采集", "处理")
flow.add_edge("处理", "分析")

# 运行
flow.run()
```
