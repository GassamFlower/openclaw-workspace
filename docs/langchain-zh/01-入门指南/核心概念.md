# LangChain 核心概念

> 掌握这些概念,你就理解了 LangChain 的工作原理

## 🎯 必知必会概念

### 1. 模型 (Model)

**模型 = LLM (大语言模型)**

LangChain 支持各种 LLM 提供商:

```python
# OpenAI GPT-4
from langchain_openai import ChatOpenAI
model = ChatOpenAI(model="gpt-4")

# Anthropic Claude
from langchain_anthropic import ChatAnthropic
model = ChatAnthropic(model="claude-sonnet-4-6")

# Google PaLM
from langchain_google_genai import ChatGoogleGenerativeAI
model = ChatGoogleGenerativeAI(model="gemini-pro")
```

**核心思想**: LangChain 提供统一的接口,你不需要关心底层 API 的差异。

---

### 2. 提示词 (Prompt)

**提示词 = 指令 + 上下文 + 输出格式**

```python
from langchain.prompts import ChatPromptTemplate

# 模板: 用占位符 {variable} 表示动态内容
template = """
你是一个{role}助手。

任务: {task}

上下文: {context}

请用{language}回答。
"""

# 填充模板
prompt = template.format(
    role="技术支持",
    task="解释什么是量子计算",
    context="量子计算是一种利用量子力学原理进行计算的技术",
    language="中文"
)
```

**提示词工程技巧**:
- 明确角色和职责
- 提供清晰的任务描述
- 给出上下文信息
- 指定输出格式
- 少样本学习

---

### 3. 链 (Chain)

**链 = 把多个步骤串起来**

Chain 就像搭积木,每个积木(组件)只做一件事,组合起来就能完成复杂任务。

```python
from langchain.chains import LLMChain

# 示例: 内容生成 + 内容检查
generate_chain = LLMChain(llm=model, prompt=generation_prompt)
check_chain = LLMChain(llm=model, prompt=checking_prompt)

# 串联
content = generate_chain.run(topic="人工智能")
final_answer = check_chain.run(content=content)
```

**常见 Chain 类型**:
| Chain 类型 | 用途 | 示例 |
|-----------|------|------|
| LLMChain | 简单的 LLM 调用 | 生成文本 |
| SequentialChain | 多步顺序执行 | 写作 + 编辑 + 格式化 |
| ConversationChain | 对话记忆 | 聊天机器人 |
| RetrievalQA | RAG 检索问答 | 文档问答系统 |

---

### 4. 智能体 (Agent)

**Agent = 会自主决策的 AI 助手**

Agent 比 Chain 更智能,因为它能:
- 观察用户输入
- 判断需要什么工具
- 自动选择工具
- 调用工具获取信息
- 综合所有信息回答

```python
from langchain.agents import create_agent

# 创建 Agent
agent = create_agent(
    model="gpt-4",
    tools=[search_tool, calculator, database_tool],
)

# Agent 自动决策
response = agent.invoke("旧金山的天气多少度? 还要计算一下换算成摄氏度")
# Agent 会: 1. 先调用 search_tool 查天气 → 2. 调用 calculator 换算 → 3. 回答用户
```

**Agent 核心流程**:
```
用户输入 → Agent 思考 → 选择工具 → 调用工具 → 观察 → 继续思考 → 回答
```

---

### 5. 工具 (Tool)

**工具 = 外部功能封装**

Agent 可以调用各种工具,让它变得更强大:

| 工具类型 | 示例 | 用途 |
|---------|------|------|
| 搜索工具 | Google Search | 网络搜索 |
| 计算工具 | Calculator | 数学计算 |
| 数据库工具 | SQL Database | 查询数据 |
| API 工具 | Weather API | 天气查询 |
| 文件工具 | File Reader | 读取文件 |
| 自定义工具 | 你的 Python 函数 | 特殊业务逻辑 |

```python
# 定义自定义工具
from langchain.tools import tool

@tool
def search_database(query: str) -> str:
    """在数据库中搜索用户信息"""
    # 实现你的逻辑
    return result

# 将工具添加到 Agent
agent = create_agent(
    model="gpt-4",
    tools=[search_database],
)
```

### 6. 检索器 (Retriever)

**检索器 = 从数据中查找相关信息**

用于 RAG (检索增强生成),让 LLM 能基于知识库回答。

```python
from langchain_community.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings

# 创建向量数据库
vectorstore = FAISS.from_texts(
    texts=["文档内容1", "文档内容2", "文档内容3"],
    embedding=OpenAIEmbeddings(),
)

# 创建检索器
retriever = vectorstore.as_retriever()

# 检索相关文档
docs = retriever.invoke("什么是 LangChain?")
# 返回与问题相关的文档片段
```

**检索器工作原理**:
```
用户问题 → 向量化 → 在向量库中查找相似文档 → 返回相关内容
```

---

### 7. 记忆 (Memory)

**记忆 = 对话历史管理**

让 Agent 记住之前的对话,实现多轮对话。

```python
from langchain.memory import ConversationBufferMemory

# 创建记忆
memory = ConversationBufferMemory()

# 存储对话
memory.save_context(
    {"input": "我的名字叫小明"},
    {"output": "你好,小明!"}
)

# 查询记忆
print(memory.load_memory_variables({}))
# 输出: {'history': 'Human: 我的名字叫小明\nAI: 你好,小明!'}
```

**常见 Memory 类型**:
| Memory 类型 | 保留内容 | 适用场景 |
|------------|---------|---------|
| ConversationBufferMemory | 完整对话历史 | 短对话 |
| ConversationBufferWindowMemory | 最近 N 轮对话 | 长对话 |
| ConversationSummaryMemory | 对话摘要 | 长对话 |
| ConversationSummaryBufferMemory | 摘要 + 最近内容 | 超长对话 |

---

### 8. 检查点 (Checkpoint)

**检查点 = 保存 Agent 状态**

支持从检查点恢复 Agent,实现:
- 持久化运行
- 中断后恢复
- 多次运行同一状态

```python
from langgraph.checkpoint.memory import MemorySaver

# 创建检查点存储
memory = MemorySaver()

# Agent 使用检查点
agent = create_agent(
    model="gpt-4",
    tools=[...],
    checkpointer=memory,
)

# 运行并保存状态
config = {"configurable": {"thread_id": "123"}}
result = agent.invoke("帮我查天气", config=config)

# 可以从中断处恢复
result2 = agent.invoke("那纽约呢?", config=config)
```

## 🎓 学习建议

**初学者路径**:
1. Model → Prompt → Chain (理解基本概念)
2. Agent → Tool (理解 Agent 如何工作)
3. Memory → Retriever (理解状态管理)

**进阶路径**:
4. Checkpoint (持久化)
5. 自定义 Tool (扩展功能)
6. 复杂 Chain 编排 (组合多个组件)

**实战路径**:
- 用 Chain 构建简单应用
- 用 Agent 实现自主决策
- 用 Retriever 做 RAG
- 用 Memory 实现多轮对话
