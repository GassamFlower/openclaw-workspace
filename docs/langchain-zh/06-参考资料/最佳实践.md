# 最佳实践 - LangChain 使用经验总结

> 从实战中总结的经验

## 💡 核心原则

### 1. 简单至上

```python
# ✅ 简单的 Chain
chain = LLMChain(llm=llm, prompt=prompt)

# ❌ 过于复杂的 Chain
complex_chain = Chain(
    step1=..., step2=..., step3=..., ...
)
`````

### 2. 明确的角色

```python
# ✅ 明确的系统提示词
system_prompt = "你是一个专业的法律咨询助手"

# ❌ 模糊的提示
system_prompt = "你是一个助手"
`````

### 3. 合理的工具

```python
# ✅ 精简的工具集
tools = [tool1, tool2]  # 只用最必要的

# ❌ 工具过多
tools = [tool1, tool2, tool3, tool4, tool5, ...]
``````

---

## 🎯 常见错误

### 错误 1: Agent 不知道用什么工具

```python
# ❌ 工具描述不清晰
Tool(name="Search", func=search, description="搜索")

# ✅ 清晰的工具描述
Tool(name="Search", func=search, description="搜索信息,用于回答关于技术、产品等问题")
`````

### 错误 2: Prompt 过于模糊

```python
# ❌ 模糊的 Prompt
prompt = "解释这个问题"

# ✅ 明确的 Prompt
prompt = "用 3-5 句话解释这个概念,重点讲清楚原理和应用"
``````

### 错误 3: 忽视错误处理

```python
# ❌ 没有错误处理
result = chain.run("data")

# ✅ 添加错误处理
try:
    result = chain.run("data")
except Exception as e:
    logger.error(e)
    result = fallback_chain.run("data")
``````

## 📊 性能优化

### 1. 使用缓存

```python
from langchain.cache import InMemoryCache
set_llm_cache(InMemoryCache())
`````

### 2. 并行执行

```python
from langchain_core.runnables import RunnableParallel

RunnableParallel(chain1=chain1, chain2=chain2).run(input="data")
``````

### 3. 选择合适的模型

```python
# 复杂任务用 GPT-4
llm = ChatOpenAI(model="gpt-4")

# 简单任务用 GPT-3.5
llm = ChatOpenAI(model="gpt-3.5-turbo")
``````
