# 常见问题 FAQ

## 📖 LangChain 常见问题

### 基础问题

**Q: LangChain 和 LangGraph 有什么区别?**

A:
- **LangChain**: 是一个更通用的 AI 应用开发框架,提供 Agent、Chain、Tool 等核心组件
- **LangGraph**: 是 LangChain 的底层框架,用于编排复杂的 AI 工作流和 Agent 行为

**Q: 如何选择 LLM 提供商?**

A:
- **GPT-4**: 复杂任务,需要高质量回答
- **GPT-3.5**: 简单任务,成本敏感
- **Claude**: 代码生成,长文本处理
- **本地模型**: 数据隐私,成本控制

**Q: LangChain 支持哪些编程语言?**

A: 主要支持 Python,也有 JavaScript/TypeScript 版本

---

### Agent 相关

**Q: Agent 和普通 Chain 有什么区别?**

A:
- **Chain**: 按顺序执行固定的步骤
- **Agent**: 可以根据问题自主决定下一步操作,包括调用工具

**Q: 如何提高 Agent 的准确性?**

A:
1. 明确工具描述
2. 提供丰富的知识库
3. 设置合理的 temperature
4. 使用 LangSmith 调试

**Q: Agent 的工具调用失败怎么办?**

A:
1. 添加错误处理
2. 提供备用工具
3. 使用重试机制
4. 记录错误日志

---

### RAG 相关

**Q: RAG 适用于什么场景?**

A: RAG 适用于需要基于特定知识库回答的场景,如:
- 企业知识库问答
- 客服系统
- 文档查询

**Q: 如何优化 RAG 的检索效果?**

A:
1. 优化文档分块策略
2. 使用更合适的向量数据库
3. 调整检索参数 (k, score_threshold)
4. 混合检索 (关键词 + 向量)

**Q: RAG 什么时候会失败?**

A:
1. 知识库不包含相关内容
2. 检索到不相关的文档
3. LLM 无法根据文档回答

### 性能优化

**Q: 如何减少 LLM 调用次数?**

A:
1. 使用缓存
2. 批量处理
3. 并行执行
4. 精简 Prompt

**Q: 如何降低 Token 使用成本?**

A:
1. 选择合适的模型 (GPT-3.5 vs GPT-4)
2. 限制输出长度
3. 使用缓存
4. 合并相似请求

**Q: 响应时间慢怎么办?**

A:
1. 使用本地模型
2. 减少检索文档数量
3. 使用流式输出
4. 优化 Prompt

## 🛠️ 常见错误

### 错误 1: Agent 不知道调用工具

**问题**:
```
Agent 一直回答"我不知道",从不调用工具
```

**解决**:
```python
# ✅ 清晰的工具描述
Tool(
    name="Search",
    func=search_function,
    description="搜索信息,用于回答关于技术、产品、服务等问题"
)
```

### 错误 2: 检索结果不相关

**问题**:
```
RAG 返回的文档与问题无关
```

**解决**:
```python
# ✅ 优化向量数据库
vector_store = FAISS.from_texts(
    texts=cleaned_documents,  # 先清洗文档
    embedding=OpenAIEmbeddings(),
)
```

### 错误 3: 记忆丢失

**问题**:
```
多轮对话中忘记上下文
```

**解决**:
```python
# ✅ 使用窗口记忆
memory = ConversationBufferWindowMemory(k=4)
```

### 错误 4: 响应太长

**问题**:
```
LLM 返回的内容过长
```

**解决**:
```python
# ✅ 限制输出长度
llm = ChatOpenAI(max_tokens=500)
```
