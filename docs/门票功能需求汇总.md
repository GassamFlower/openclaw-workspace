## 📋 需求概述

微信小程序门票功能需要进一步优化，支持多场馆权益核销、双端记录同步、超级管理员多场景核销、核销员权限绑定等。

## 🎯 需求清单

### 需求 1：多场馆权益核销
**状态**: ✅ 已完成
**优先级**: 高
**工期**: 2-3天

**问题描述**:
> 门票包含了几个场馆权益，就需要核销几次才能使用

**需求描述**:
- 一张门票可能包含多个场馆的权益（如：主馆+副馆+临时展馆）
- 用户需要依次进入每个场馆核销
- 系统需要记录每个场馆的核销状态
- 核销完成所有场馆后，门票才能完全使用

**功能要求**:
1. 门票显示包含的场馆列表
2. 用户进入场馆后可核销对应权益
3. 系统记录每个场馆的核销状态
4. 未核销的场馆不可进入
5. 所有场馆核销完成后，门票状态变为"已使用"

**示例场景**:
```
门票类型：通票（主馆 + 副馆 + 临时展馆）

用户操作流程：
1. 进入主馆 → 扫码核销主馆权益 → 状态：已核销
2. 进入副馆 → 扫码核销副馆权益 → 状态：已核销
3. 进入临时展馆 → 扫码核销临时展馆权益 → 状态：已核销
4. 门票状态：完全使用 ✅
```

**技术要点**:
- 门票表增加 `venue_count` 字段
- 核销表记录 `venue_id` 和 `venue_index`
- 门票表增加 `venues_used` 字段（记录已核销场馆）
- 门票表增加 `venue_status` 字段（场馆核销状态：0=未核销, 1=已核销）

### 需求 2：双端核销记录同步
**状态**: 🔄 待开发
**优先级**: 高
**工期**: 2-3天
**依赖**: 需求1

**问题描述**:
> 核销记录需要在小程序和web端都有记录

**需求描述**:
- 核销记录需要在多个端同步显示
- 用户端：小程序端查看自己的核销记录
- 管理端：Web端查看所有核销记录
- 订单端：小程序和Web端查看订单状态

**功能要求**:
1. 核销操作在任一端执行，其他端同步更新
2. 小程序端：
   - 用户订单列表显示核销状态
   - 每个场馆的核销状态显示
3. Web管理端：
   - 订单管理页面显示核销记录
   - 核销时间、场馆、操作人
   - 核销统计
4. 实时同步机制

**技术要点**:
- 使用 WebSocket 实现实时同步
- 核销操作后触发 Webhook 推送
- 数据库事务保证一致性
- Redis 缓存同步核销状态

### 需求 3：超级管理员多场景核销
**状态**: 🔄 待开发
**优先级**: 中
**工期**: 1-2天
**依赖**: 需求1

**问题描述**:
> 超级管理员有核销所有票的权限，但也是有几个场馆就需要核销几次，也就是扫码过手动核销几次

**需求描述**:
- 超级管理员具备核销所有门票的权限
- 超级管理员扫码核销后，需要手动确认核销
- 如果门票包含多个场馆，超级管理员需要依次核销每个场馆
- 核销记录需要记录操作人、时间、场馆

**功能要求**:
1. 超级管理员扫码后：
   - 显示门票信息
   - 显示包含的场馆列表
   - 显示各场馆核销状态
2. 手动核销确认：
   - 超级管理员选择要核销的场馆
   - 点击"核销"按钮确认
   - 系统记录核销操作
3. 核销记录：
   - 操作人：超级管理员
   - 核销时间：自动记录
   - 核销场馆：手动选择
   - 订单号：关联订单

**示例场景**:
```
超级管理员操作流程：

1. 扫描门票二维码
   → 显示：通票（主馆 + 副馆 + 临时展馆）
   → 当前状态：未核销

2. 选择主馆 → 点击"核销"
   → 记录：超级管理员，主馆，2026-03-26 15:00:00

3. 选择副馆 → 点击"核销"
   → 记录：超级管理员，副馆，2026-03-26 15:01:00

4. 选择临时展馆 → 点击"核销"
   → 记录：超级管理员，临时展馆，2026-03-26 15:02:00

5. 门票状态：完全使用 ✅
```

**技术要点**:
- 超级管理员权限验证
- 核销确认机制（二次确认）
- 核销记录日志
- 门票状态更新逻辑

### 需求 4：核销员权限绑定
**状态**: 🔄 待开发
**优先级**: 高
**工期**: 1-2天
**依赖**: 需求1

**问题描述**:
> 核销员绑定过场馆权益，核销员扫同一个码也只能核销自己所在的场馆

**需求描述**:
- 核销员需要绑定授权的场馆权限
- 核销员扫码后只能核销自己绑定的场馆
- 防止核销员跨场馆核销
- 超级管理员可以核销所有场馆

**功能要求**:
1. 核销员权限配置：
   - 管理员可以为核销员绑定场馆权限
   - 一个核销员可以绑定多个场馆
   - 核销员只能核销已授权的场馆
2. 核销验证：
   - 核销员扫码后，系统验证场馆权限
   - 如果权限不匹配，提示错误信息
   - 只有授权场馆才可核销
3. 权限管理：
   - 超级管理员可查看所有核销员权限
   - 可修改核销员权限
   - 可查看核销记录

**示例场景**:
```
核销员权限配置：

管理员操作：
- 为核销员A绑定：主馆、副馆
- 为核销员B绑定：临时展馆

核销员A扫码：
- 扫描主馆二维码 → ✅ 核销成功（已授权）
- 扫描副馆二维码 → ✅ 核销成功（已授权）
- 扫描临时展馆二维码 → ❌ 无权限（未授权）

核销员B扫码：
- 扫描临时展馆二维码 → ✅ 核销成功（已授权）
- 扫描主馆二维码 → ❌ 无权限（未授权）

超级管理员扫码：
- 扫描任何场馆 → ✅ 核销成功（所有权限）
```

**技术要点**:
- 核销员表增加 `venue_ids` 字段（JSON数组）
- 核销操作时验证权限
- 超级管理员特殊权限标记
- 权限缓存机制
