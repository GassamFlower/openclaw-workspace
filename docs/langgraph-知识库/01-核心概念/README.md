# 01 - 核心概念

> LangGraph 的核心思想，理解这些才能用好它

## 什么是 LangGraph？🧭

简单说：**一个低级别的 Agent 编排框架和运行时**

- **目标**：让你能够创建复杂的、确定性和智能化的工作流
- **核心**：基于图（Graph）的工作流编排
- **特点**：
  - 可视化 Graph 设计
  - State 管理和持久化
  - 支持确定性和智能工作流
  - 人类在回路上（Human-in-the-Loop）
  - 实时流式输出

**谁适合用？**
- 需要高度自定义的 Agent
- 复杂的工作流编排
- 需要确定性行为的场景

---

## 它是如何工作的？

```
状态（State） → 节点（Node） → 边（Edge） → 状态（State）
```### 核心概念

1. **Graph** - 图，工作流的整体结构
2. **Node** - 节点，工作流中的执行单元
3. **Edge** - 边，节点间的连接
4. **State** - 状态，数据存储和传递
5. **Checkpointer** - 检查点，状态持久化

---

## LangGraph vs 其他方案

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **LangGraph** | 低级别编排框架 | 复杂工作流、高度自定义 |
| **LangChain Agents** | 高级别 Agent 框架 | 快速构建 Agent |
| **Deep Agents** | 内置高级特性的 Agent | 现代特性、开箱即用 |

## 为什么选择 LangGraph？

### 优势

- ✅ **灵活性高** - 支持完全自定义
- ✅ **确定性强** - 可预测的行为
- ✅ **可视化设计** - 拖拽式设计
- ✅ **状态持久化** - 支持断点续传

### 适用场景

✅ **适合**：
- 复杂的业务流程
- 需要精确控制的场景
- 多步骤的 Agent 任务

❌ **不适合**：
- 简单的聊天机器人
- 快速原型开发
- 不需要自定义的场景

## 核心概念速查

| 概念 | 简单解释 |
|------|----------|
| **Graph** | 工作流结构，节点和边的集合 |
| **Node** | 工作流中的执行单元（函数） |
| **Edge** | 节点间的连接 |
| **State** | 数据状态，在工作流中传递 |
| **Checkpointer** | 状态持久化存储 |
| **Router** | 路由节点，决定下一个执行节点 |
