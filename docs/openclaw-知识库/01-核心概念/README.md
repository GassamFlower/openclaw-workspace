# 01 - 核心概念

> OpenClaw 的核心思想，理解这些才能用好它

## 什么是 OpenClaw？🦞

简单说：**一个"AI 代理 + 多渠道网关"系统**

- **目标**：让 AI 代理能通过各种聊天软件（WhatsApp、Telegram、Discord、iMessage 等）和你交流
- **核心**：你在自己的机器上运行一个 Gateway，它就是"桥梁"
- **特点**：
  - 自托管（跑在你自己的电脑/服务器上）
  - 多渠道（一个 Gateway 服务所有聊天软件）
  - Agent 原生（专为 AI 代理设计，支持工具调用、会话管理、多 Agent 路由）
  - 开源（MIT 许可）

**谁适合用？**
- 开发者
- 想要个人 AI 助手，但不信任云服务的用户
- 需要远程访问 AI 的场景

---

## 它是如何工作的？

```
[你的聊天软件] ←→ [Gateway] ←→ [AI 代理]
                   ↑
              你的电脑/服务器
```

### 关键组件

1. **Gateway**（网关）
   - 核心服务，运行在机器上
   - 处理所有消息路由
   - 管理会话、工具调用、权限

2. **AI Agent**（AI 代理）
   - 负责思考和执行任务
   - 支持 Pi、Claude、GPT 等模型
   - 可以使用各种工具（搜索、代码执行、文件操作等）

3. **Node**（节点）
   - 移动设备应用（iOS、Android）
   - 增强功能：摄像头、位置、语音唤醒
   - 通过安全通道连接到 Gateway

4. **Control UI**（控制面板）
   - 浏览器界面
   - 查看消息、配置、管理会话
   - 默认地址：http://127.0.0.1:18789/

## 核心能力一览

| 功能 | 说明 | 例子 |
|------|------|------|
| **多渠道** | WhatsApp、Telegram、Discord、iMessage 等同时接入 | 在 Telegram 发消息，在 Discord 收回复 |
| **插件扩展** | 支持第三方插件（Mattermost 等） | 安装插件支持 Matrix、IRC |
| **多 Agent 路由** | 每个 Agent 独立会话 | 给 Pi 开发任务，给 Claude 写报告，互不干扰 |
| **媒体支持** | 图片、音频、文档 | 发送截图，接收语音消息 |
| **Web 控制面板** | 浏览器管理界面 | 在网页上看聊天记录、配置参数 |
| **移动节点** | iOS/Android 手机增强功能 | 用手机摄像头拍照传给 AI |

## 为什么选择 OpenClaw？

### 对比云服务方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **OpenClaw（自托管）** | 数据在自己机器上<br>完全控制<br>可定制 | 需要自己维护 |
| **云服务（如 ChatGPT Plus）** | 开箱即用<br>无需维护 | 数据在云端<br>功能受限 |

### 适用场景

✅ **适合**：
- 想要数据隐私
- 需要远程访问 AI
- 开发者/技术爱好者
- 多设备协同工作

❌ **不适合**：
- 没有技术能力维护服务器
- 只想简单用用 AI 聊天
- 对隐私要求不高

## 核心概念速查

| 概念 | 简单解释 |
|------|----------|
| **Session** | 会话 - 一个聊天上下文 |
| **Channel** | 渠道 - 具体的聊天软件（WhatsApp、Telegram 等） |
| **Agent** | 代理 - AI 本身 |
| **Workspace** | 工作区 - Agent 的工作目录 |
| **Plugin** | 插件 - 扩展功能（如新渠道） |
| **Node** | 节点 - 移动设备 |
